# Project Goal (Unchanging)

**pixel-office 프로젝트의 목표는 "코딩 에이전트(Claude Code 등)의 상태를 시각적으로 화면에 표시하는 것"입니다.**

이 목표는 기술 스택/엔진/맵 포맷이 바뀌어도 변하지 않습니다. 모든 설계/구현 결정은 이 목표를 우선합니다.

## Implications

- **화면 속 캐릭터 = 에이전트 인스턴스**. 걷기/타이핑/앉기/idle 애니메이션은 실제 에이전트의 상태(작업 중, 도구 호출 중, 대기 중 등)를 반영합니다. 게임적 재미가 아닌 "관찰 가능성(observability)"이 목적입니다.
- **오피스는 ambient 컨텍스트**. 맵/가구/좌석은 에이전트의 상태와 소속을 공간적으로 표현하는 수단입니다.
- **React/VS Code 통합이 1순위**. 이 앱은 독립 게임이 아니라 IDE/툴 사이드 패널에 임베드되는 상태 시각화 도구입니다. 엔진 선택도 이 경계에서 평가되어야 합니다.
- **에이전트 상태 → 시각 상태 매핑이 핵심 파이프라인**. pathfinding, colorize, 좌석 배정, 사회적 버블 같은 기존 로직은 이 매핑의 일부입니다. 엔진 교체 시 이 로직을 손실 없이 유지하는 것이 최우선 제약입니다.

## Non-goals

- 게임플레이, 유저 플레이어블 조작, 퍼즐, RPG 시스템
- 전투, 물리, 사운드 효과 (적어도 현재 범위 밖)
- 수많은 엔티티를 위한 극한 성능 최적화 (규모가 작음: 수십 캐릭터/타일 200~2000)
